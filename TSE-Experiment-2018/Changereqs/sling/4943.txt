[SLING-4943] ResourceNotFoundException: Include resource path in the getMessage() method - ASF JIRA
Currently if instantiating a ResourceNotFoundException with both a resource path and a message only the message is given out.
That is e.g.
a problem in DefaultGetServlet (https://github.com/apache/sling/blob/trunk/bundles/servlets/get/src/main/java/org/apache/sling/servlets/get/impl/DefaultGetServlet.java#L218).
